What the patterns mean

  • Key drift is the dominant issue. Most flows speak the correct text but store it under a key the library doesn't define — e.g. Exception_Unexpected / Exception_Vocalls where the registry names it Scheduler_ExceptionDisconnect, or a placeholder unknown. The audio is right; the key label is not canonical.
  • PreQueue_PreQueue has no source in these libraries. The line “U wordt zo dadelijk doorverbonden” exists in the registry only under other leaves (MELDJEAAN/Adhoc_Transfer, SV/Menu_Transfer). The DL, PM, LEZ, TS and MIGRATIE libraries carry no play-prompt for it — flagged as a gap, left as-is per your instruction.
  • Empty text is the only hard failure. Eight AdHoc_Message ops (PM, SW-LEZ, LEZ-BACKOFFICE) have an empty NL string — nothing plays. Some map to a real registry AdHoc_* entry; others are genuinely blank ad-hoc slots.
  • One corrupted character originates upstream. The MIGRATIE-2DELIJN closed message reads “cli☒nt” — but the registry markdown itself is corrupted the same way, so this is a source-data fix, not a flow-only one.
  • MPA uses the GAPA registry. Its Info_Choice_* keys correspond to GAPA Menu_Choice_*; its exception text matches Scheduler_EmergencyDisconnect, not Emergency_Emergency.
